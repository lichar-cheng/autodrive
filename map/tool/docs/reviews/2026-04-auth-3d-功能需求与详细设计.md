# AutoDrive 扫图工具 V0.6 新功能评审材料
## （功能需求说明 + 开发详细设计）

- 版本：V0.6
- 文档类型：公司内部评审（需求 + 详细设计）
- 适用范围：`map/tool`（server、client、client_desktop）

---

## 1. 背景与目标

现网版本以固定后端直连 + 2D 扫图为主，在以下方面存在明显短板：

1. **接入方式单一**：无法满足“现场局域网直连”和“云端统一认证”并存的项目部署诉求。
2. **数据维度受限**：仅支持 `LaserScan` 2D 建图，无法满足 3D 点云预览与保存需求。
3. **流程割裂风险**：客户端若分别适配 local/cloud、2D/3D，后续演进成本和故障概率会快速上升。

本版本目标：

- 新增 `local | cloud` 双认证模式，并统一归一为后端连接描述符；
- 新增 `2d | 3d` 扫图模式选择，2D 与 3D 流程互斥运行；
- 在 3D 模式下实现“可交互预览 + 有界保存”，避免前端卡顿和文件爆炸；
- 扩展 `.slam` 为模式感知保存格式，兼容 2D 资产并承载 3D 点云。

---

## 2. 范围定义

### 2.1 本版本范围（In Scope）

1. 认证模式选择：`local` / `cloud`。
2. 登录成功后统一输出连接描述符：`backend_host`、`backend_port`、`token`、`expires_at`。
3. 扫图模式选择：`2d` / `3d`。
4. 2D 流程：延续现有 LaserScan 累计与编辑能力。
5. 3D 流程：支持 PointCloud2 预览、统计、保存、加载。
6. `.slam` 模式化归档：2D 使用 `radar_points.bin`，3D 使用 `point_cloud.bin`。

### 2.2 非本版本范围（Out of Scope）

1. 2D 与 3D 同时采集并行融合。
2. 重型 3D 编辑（切片、分割、重建、语义标注）。
3. 原始全量点云历史归档。
4. 云端链路的全程代理转发（本版仅涵盖认证和后端解析）。

---

## 3. 功能需求说明（FRD）

## 3.1 功能一：双认证模式

### 3.1.1 用户价值

- 现场工程人员可以直接使用目标设备登录接口（local）；
- 平台化项目可以走云端统一认证（cloud）；
- 无论走哪种认证，下游连接逻辑都保持统一，降低运维复杂度。

### 3.1.2 业务规则

1. 认证模式仅允许 `local` 或 `cloud`；非法值回退 `local`。
2. local 登录默认路径为 `/login`；cloud 默认路径为 `/api/auth/login`。
3. local 默认用户名/密码支持环境变量覆盖。
4. 登录返回可能存在字段差异，客户端需兼容 `host/ip/backend_host`、`port/backend_port`、`token/access_token/auth_token` 等常见别名。
5. 登录成功后产出统一描述符，所有 HTTP/WS 通信均基于描述符派生。

### 3.1.3 输入/输出

- 输入：认证模式 + 认证参数（IP/Host、Port、Path、账号密码）。
- 输出：

```json
{
  "auth_mode": "local",
  "backend_host": "10.0.0.12",
  "backend_port": 8080,
  "token": "abc",
  "expires_at": null
}
```

### 3.1.4 验收标准

1. 可在浏览器端、桌面端切换认证模式并完成登录。
2. 登录成功后可自动解析并填充后端 WS 地址。
3. 后续 HTTP 请求自动带 Bearer Token。
4. 认证失败时有可读错误信息和日志。

---

## 3.2 功能二：2D/3D 扫图模式切换

### 3.2.1 用户价值

- 2D 场景保持高效扫图与编辑；
- 3D 场景可快速预览点云并存档，为后续算法或离线处理提供数据基础。

### 3.2.2 业务规则

1. 扫图模式仅允许 `2d` 或 `3d`；非法值回退 `2d`。
2. 同一时刻仅允许一个模式激活（互斥）。
3. 切换模式会重置当前扫描会话，避免数据串模。
4. 服务端与客户端模式必须保持一致（客户端切换后调用 `/scan/mode`）。

### 3.2.3 模式行为

- **2D 模式**：订阅前/后激光主题，生成占据栅格与 2D 点；保存为 `radar_points.bin`。
- **3D 模式**：订阅 PointCloud2，服务端降采样后发布 `/pointcloud/preview`；保存为 `point_cloud.bin`。

### 3.2.4 验收标准

1. 模式切换后，2D 与 3D 数据流不会交叉处理。
2. 3D 预览在常规笔记本上可保持交互流畅（不出现明显假死）。
3. 保存后的 `.slam` 能正确反映 `scan_mode` 且可被加载。

---

## 3.3 功能三：3D 有界预览与有界保存

### 3.3.1 用户价值

在保证可视化可用性的前提下控制 CPU、内存、网络和存储开销。

### 3.3.2 约束与策略

1. 原始点云不直接全量推送给前端。
2. 预览层采用体素降采样 + 点数上限控制。
3. 保存层采用比预览更高保真但仍受限的点云集合。
4. WebSocket 再次限流（服务端发送前可二次抽样）。

### 3.3.3 关键指标（默认值）

- `pointcloud_preview_voxel_size = 0.12`
- `pointcloud_save_voxel_size = 0.06`
- `pointcloud_preview_limit = 5000`
- `pointcloud_save_limit = 50000`
- `POINTCLOUD_MAX_WS_POINTS = 3000`

### 3.3.4 验收标准

1. 单帧预览点数不超过配置上限。
2. 3D 保存后点数受限且文件体积可控。
3. 高负载情况下客户端仍可持续收到关键帧。

---

## 3.4 功能四：`.slam` 模式化存储

### 3.4.1 用户价值

用统一扩展名承载 2D/3D，不增加运维认知负担，同时为后续版本演进保留格式空间。

### 3.4.2 存储规则

1. 2D：`manifest.json + radar_points.bin`。
2. 3D：`manifest.json + point_cloud.bin`。
3. `manifest.json` 必须包含 `scan_mode`。
4. 版本号：2D 使用 `stcm.v2`，3D 使用 `stcm.v3`。

### 3.4.3 验收标准

1. 2D 文件加载后仍维持原有行为。
2. 3D 文件加载后能恢复点云点集与模式。
3. 导出链路（PGM/YAML/JSON）不受破坏。

---

## 4. 非功能需求（NFR）

1. **性能**：点云处理和 WS 下发需保证客户端可交互。
2. **稳定性**：断连自动重连、队列近满告警、超时踢除空闲连接。
3. **兼容性**：保留 2D 既有能力与老地图导入路径。
4. **可观测性**：健康检查返回 mapping 前置条件、scan summary、网络状态。
5. **安全性（基础）**：token 仅在会话中使用，不写入明文地图文件主体字段。

---

## 5. 开发详细设计（DDS）

## 5.1 总体架构

### 5.1.1 模块边界

1. **client_desktop（Tk）**：主交付端，含认证 UI、扫描控制、地图编辑、保存加载。
2. **client（Browser）**：调试基线，实现同构认证与模式切换行为。
3. **server（FastAPI + ROS）**：模式切换、数据汇聚、预览下发、地图编解码。

### 5.1.2 关键设计思想

- “多入口、单出口”：local/cloud 认证统一归一为连接描述符；
- “单模式激活”：2D/3D 明确互斥，流程上天然避免串扰；
- “分层点云处理”：原始层、预览层、保存层职责分离。

---

## 5.2 认证详细设计

### 5.2.1 数据模型

- `AuthMode = Literal["local", "cloud"]`
- `BackendConnectionDescriptor`
  - `auth_mode`
  - `backend_host`
  - `backend_port`
  - `token`
  - `expires_at`

### 5.2.2 核心流程

1. 用户选择认证模式并提交表单。
2. 客户端按模式拼装登录 URL（含默认 path）。
3. 解析响应（兼容多字段别名）。
4. 调用归一化函数得到描述符。
5. 基于描述符推导：
   - HTTP Base（`http/https://host:port`）
   - WS URL（`ws/wss://host:port/ws/stream`）
6. Token 写入 HTTP Session Header，驱动后续 API 调用。

### 5.2.3 异常处理

- host 为空、port 非法、字段缺失时直接失败并提示；
- 请求超时按 8s 截断，避免 UI 长时间无响应；
- 认证失败保留原连接状态，不覆盖已有可用连接配置。

---

## 5.3 扫图模式详细设计

### 5.3.1 状态机

- 状态字段：`scan_mode`、`scan_active`、`scan_summary`。
- 状态迁移：
  1. `POST /scan/mode` 切换模式。
  2. 服务端更新 `CONFIG.scan_mode` 与 `CONFIG.ros.scan_mode`。
  3. 重置扫描会话（帧计数、累计点、时间戳）。

### 5.3.2 2D 流

1. 处理 `/lidar/front` `/lidar/rear`。
2. 按体素网格累计 `x/y/intensity` 与命中次数。
3. UI 继续提供擦除、障碍线绘制、二阶段编辑。

### 5.3.3 3D 流

1. 处理 PointCloud2。
2. 预览点发布到 `/pointcloud/preview`。
3. 客户端仅在 `scan_mode=3d` 时消费该 topic。
4. 3D 画布为只读预览，不开放编辑。

---

## 5.4 3D 点云管线设计

### 5.4.1 三层模型

1. **Raw Ingest**：解码 PointCloud2，做瞬态处理；
2. **Preview**：大体素 + 限点，保障实时显示；
3. **Save**：小体素 + 更高上限，用于落盘。

### 5.4.2 限流与削峰

- 主题最小发送间隔控制；
- 队列接近上限时告警并统计；
- WebSocket 出口抽样，避免单客户端拖垮服务。

### 5.4.3 诊断项

- `scan_summary.pointcloud_frames`
- `mapping_prereq`（ready/blockers/warnings）
- `/diag/stream_stats` 队列压力与丢帧统计

---

## 5.5 `.slam` 编解码设计

### 5.5.1 保存接口

- `POST /map/save`
- 入参：`name`、`notes`、`voxel_size`、`scan_mode`、`reset_after_save`
- 逻辑：按模式写入 `radar_points` 或 `point_cloud`，并落地 `manifest`。

### 5.5.2 加载接口

- `POST /map/load`
- 逻辑：读取 `scan_mode`，选择对应点集合回填。
- 2D 加载时重建累计栅格索引；3D 加载保持点云集合。

### 5.5.3 兼容策略

- 无 `scan_mode` 字段默认按 2D 处理；
- 2D 与 3D 均允许“空点兜底”（保存最小占位点）避免坏档。

---

## 5.6 接口设计总表（增量）

1. `POST /scan/mode`
   - 请求：`{ "scan_mode": "2d|3d" }`
   - 响应：`ok + scan_mode + scan_summary + mapping_prereq`
2. `POST /map/save`
   - 请求支持 `scan_mode`
   - 响应增加 `contains.point_cloud` 与 `scan_mode`
3. `POST /map/load`
   - 响应返回 `scan_mode` 和 `point_count`
4. `GET /health`
   - 返回 mapping readiness、scan summary

---

## 5.7 客户端交互设计

### 5.7.1 桌面端

1. 顶部新增认证模式切换与 local/cloud 参数区。
2. 扫图区新增 `Scan Mode` 下拉框。
3. 3D 模式下显示点云预览统计，禁用 2D 编辑语义按钮。
4. 保存/加载时自动带当前模式并反馈点数摘要。

### 5.7.2 浏览器端

1. 与桌面端保持同构字段和流程。
2. 3D 仅处理 `/pointcloud/preview`，2D 仅处理激光 topic。
3. 认证成功后自动刷新连接状态提示。

## 6. 测试与验收方案

## 6.1 单元测试（已规划）

1. 认证描述符归一化（local/cloud、字段别名、端口校验）。
2. 扫图模式默认值与非法值回退。
3. PointCloud 预览限点、模式互斥处理。
4. `.slam` 2D/3D 保存加载行为。

## 6.2 集成回归

1. 本地/云认证 -> 连接 -> 扫描 -> 保存 -> 加载全链路。
2. 2D 模式下编辑功能不回归。
3. 3D 模式下长时间预览无明显内存泄漏。

## 6.3 现场验收建议

1. 使用真实认证接口进行一次 local/cloud 双链路验收。
2. 以 10 分钟以上轨迹执行 3D 压测，观察 UI 流畅度和文件大小。
3. 抽检 2D 历史 `.slam` 文件导入兼容性。

---

## 7. 风险与缓解

1. **认证接口字段不统一**
   - 缓解：继续扩展字段别名映射 + 增加可配置解析器。
2. **3D 点云场景差异大，参数一刀切困难**
   - 缓解：暴露 preview/save 体素与限点配置，提供项目级默认模板。
3. **模式切换时用户误操作导致数据丢失感知**
   - 缓解：切换前增加“未保存数据提醒”与确认弹窗。

---

## 8. 里程碑与发布建议

1. M1：认证 + 描述符贯通（桌面/浏览器）。
2. M2：scan_mode 接口和客户端模式同步。
3. M3：3D 预览有界化 + `.slam` 3D 存储。
4. M4：回归测试 + 文档冻结 + 发布。

建议发布策略：

- 先灰度到研发/测试环境，观测 3D 预览稳定性；
- 一周无阻断问题后全量切换。

---

## 9. 评审关注点（建议会议逐项确认）

1. 是否接受“3D 只预览不编辑”的阶段性目标。
2. local/cloud 实际认证返回契约是否需要统一网关层。
3. 3D 默认限点与体素参数是否满足目标硬件配置。
4. 地图文件版本升级（v2/v3）是否需要离线迁移工具。
